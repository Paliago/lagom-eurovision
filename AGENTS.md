# Lagom Eurovision — Agent Notes

## Stack & Tooling

- **Package manager**: `bun` (lockfile is `bun.lock`). Use `bun run <script>`.
- **Frontend**: React 19 + Vite + TypeScript + Tailwind CSS v4.
- **Backend**: Convex (real-time DB + functions).
- **Deployment**: SST v3 (`sst.config.ts`) → AWS StaticSite in `eu-north-1`, domain `lagomeurovision.com`.
- **Router**: `react-router` v7. Import from `react-router` **only** — `react-router-dom` is not used.
- **UI components**: shadcn/ui (style "new-york", non-RSC) with `lucide-react` icons.

## Developer Commands

| Command | What it does |
|---------|--------------|
| `bun run dev` | Runs Vite frontend + Convex backend in parallel via `npm-run-all` |
| `bun run dev:frontend` | Vite dev server (`vite --open`) |
| `bun run dev:backend` | `convex dev` (watches & pushes functions) |
| `bun run build` | `tsc -b && vite build` |
| `bun run lint` | `tsc && eslint . --ext ts,tsx --report-unused-disable-directives --max-warnings 0` |
| `bun run preview` | Vite preview of production build |

**Note**: `predev` runs `convex dev --until-success && convex dashboard`, so `bun run dev` will block briefly until Convex syncs and then open the dashboard.

## Project Structure

```
src/
  main.tsx          # ConvexReactClient init + ConvexProvider + BrowserRouter
  App.tsx           # react-router routes (year-prefixed and default)
  pages/            # LandingPage, ContestantListPage, ContestantRatingPage, OverviewPage
  lib/              # contestants.ts (hardcoded data), year.ts (URL year parsing), utils, hooks
  components/ui/    # shadcn/ui components
convex/
  schema.ts         # rooms + ratings tables with indexes
  rooms.ts          # joinOrCreateRoom mutation
  ratings.ts        # submitRating, getUserRatingForContestant, getRatingsForRoomAndContestant, getOverviewRatingsForRoom
  _generated/       # Auto-generated by Convex (ignored by ESLint)
```

## Year Routing

The app supports multiple Eurovision years. The **default** year is `2026` (see `src/lib/contestants.ts`).

- Default routes: `/room/:roomName/contestants`, `/room/:roomName/contestant/:id`, `/room/:roomName/overview`
- Previous-year routes prefix with `/:year/`, e.g. `/2025/room/:roomName/contestants`
- Navigation preserves the year segment automatically.
- To add a new year: append contestant array in `src/lib/contestants.ts`, update `DEFAULT_YEAR`, add to `VALID_YEARS`.

## Convex Backend

- Functions are file-routed: a query in `convex/ratings.ts` named `submitRating` is callable as `api.ratings.submitRating`.
- Always define `args` and `returns` validators on every query/mutation. If a function returns nothing, use `returns: v.null()`.
- Use `v.id("tableName")` for ID fields; on the client these are strings.
- `convex/tsconfig.json` is separate from the frontend TS config and targets `ESNext`.
- The `_generated` directory is auto-generated and ignored by ESLint.

## Environment

- `VITE_CONVEX_URL` is required at build/dev time. It is set in `.env.local` (committed for this repo).
- `convex dev` requires the Convex CLI to be authenticated (`npx convex login`).

## Linting & Style

- ESLint ignores: `dist`, `convex/_generated`, `postcss.config.js`, `tailwind.config.js`, `vite.config.ts`, `eslint.config.js`.
- Key relaxed rules (intentional for Convex + gradual TS adoption):
  - `@typescript-eslint/no-explicit-any`: **off**
  - `@typescript-eslint/require-await`: **off** (Convex handlers often async without await)
  - `@typescript-eslint/no-unsafe-argument/assignment/call/member-access/return`: **off**
  - `@typescript-eslint/no-unused-vars`: **warn** (ignores `_` prefix)
- Tailwind CSS v4 is used with `@tailwindcss/vite` plugin (no separate PostCSS config needed).

## Important Gotchas

- **No test runner** is configured. If adding tests, install and wire up explicitly.
- **Contestant data is hardcoded** in `src/lib/contestants.ts`, not fetched from an API.
- **User/session state** is stored in `localStorage` under keys `eurovisionUserId`, `eurovisionNickname`, `eurovisionRoomId`.
- **Build artifact**: `dist/` is the Vite build output; SST deploys from there.
